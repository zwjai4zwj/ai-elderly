## 技术架构
| 组件 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | Vue 3 + Vite | H5响应式，适配微信浏览器 |
| 后端 | Netlify Functions | Serverless API |
| 数据库 | Supabase | 免费PostgreSQL |

## 开发历程

### 第一阶段：需求确认（4月6日 21:00-22:00）
- 明确功能需求：病例生成、AI对话、自动评分、班级管理
- 确定技术方案：Vue 3 + Netlify + Supabase + DeepSeek
- 用户认证：账号密码 + 班级码

### 第二阶段：第一次开发（4月6日 22:00-4月7日 05:00）
- 完整版代码开发
- 包含用户系统、班级管理、数据分析等全部功能
- 代码量较大，构建配置复杂

### 第三阶段：Vercel部署尝试（4月7日 05:00-11:00）
**遇到的问题：**
1. GitHub授权问题 - 解决：创建Token
2. 构建失败 - 缺少依赖 → 添加@vercel/node, @types/node
3. TypeScript错误 - 解决：修改tsconfig.json
4. 重复键错误 - 解决：合并types配置
5. 页面无法访问 - **Vercel在国内被限制**

**关键教训：**
- Vercel虽然免费，但国内访问不稳定
- 应优先考虑国内用户访问情况

### 第四阶段：转战Netlify（4月7日 11:00-11:50）
- 注册Netlify
- 导入GitHub仓库
- **一次成功！** 🎉

### 第五阶段：功能完善（进行中）
- 添加真实AI对话功能
- 集成DeepSeek API
- 完善评分系统

## 遇到的主要坑

### 1. Vercel国内访问问题
- **问题**：Vercel部署成功但国内无法访问
- **解决**：改用Netlify

### 2. 构建配置复杂
- **问题**：TypeScript、依赖、路径配置容易出错
- **解决**：简化配置，使用CDN引入Tailwind

- **问题**：需要GitHub Token才能推送
- **解决**：用户创建Token，我帮助推送

## 经验总结

1. **部署平台选择**：优先考虑国内访问稳定性，Netlify比Vercel更稳定
2. **代码简洁原则**：先做出能跑的版本，再逐步完善
3. **环境变量管理**：API Key等敏感信息通过环境变量传递
4. **用户沟通**：了解用户技术水平，提供详细的图文教程
